# GoDict

[English version](README.md)

GoDict — небольшое кроссплатформенное приложение для голосового ввода. Выберите шаблон, LLM и язык распознавания, нажмите **Start recording**, продиктуйте текст и нажмите кнопку ещё раз. GoDict передаёт поток PCM-аудио в Yandex SpeechKit v3, отправляет финальный текст в совместимую с OpenAI Responses LLM и копирует результат в буфер обмена.

## Настройка

1. Выберите один из стартовых файлов: `godict.ru.config`, `godict.en.config` или объединённый `godict.config`.
2. Скопируйте выбранный файл рядом с исполняемым файлом под именем `godict.config`.
3. Задайте переменные окружения, на которые ссылаются значения `%NAME%`, либо укажите учётные данные прямо в файле. Не коммитьте реальные ключи.
4. Разрешите доступ к микрофону и запустите приложение.

`godict.ru.config` — русская основная версия; `godict.en.config` — её английский перевод; `godict.config` содержит шаблоны на обоих языках. В `settings` параметр `beep_on_completion` включает короткий сигнал после успешного копирования результата и по умолчанию равен `true`.

## Установка

Распакуйте архив релиза, выберите стартовый файл конфигурации и храните его рядом с исполняемым файлом под именем `godict.config`. В macOS каталог `/usr/local/bin` обычно уже входит в `PATH`, поэтому оба файла можно установить туда:

```sh
sudo cp godict godict.config /usr/local/bin/
```

macOS-бинарники не подписаны. Если система блокирует скачанный исполняемый файл, снимите с него флаг карантина и затем запустите из Terminal:

```sh
sudo xattr -d com.apple.quarantine /usr/local/bin/godict
godict
```

## Формат конфигурации

`godict.config` использует HCL. Блоки `model` и `template` имеют имена. Укажите `default = true` ровно для одного шаблона, чтобы выбрать его при запуске. Плейсхолдеры: `{recognized_text}` — результат SpeechKit, `{language}` — выбранный код языка, `{clipboard}` — содержимое буфера обмена; без `{clipboard}` буфер не читается.

Необязательный `project` передаётся как заголовок `OpenAI-Project`. Необязательный `reasoning` передаётся как `reasoning.effort`; для нескольких уровней создайте несколько почти одинаковых блоков модели. Флажок **Web search** добавляет `tools: [{"type":"web_search"}]`; **Display result** показывает только для чтения результат, который также копируется в буфер.

## Локальная разработка и релизы

```powershell
go test ./...
go run .
.\scripts\build-release.ps1 -Version 0.1.0
```

Локальный скрипт использует Docker/Fyne-cross. GitHub Actions собирает Windows x64, Linux x64 и macOS ARM64 на нативных раннерах; `build_windows_arm64` дополнительно включает Docker-сборку Windows ARM64. При `create_release=false` ZIP-файлы остаются артефактами запуска; при `create_release=true` с заголовком и описанием создаётся GitHub Release.
